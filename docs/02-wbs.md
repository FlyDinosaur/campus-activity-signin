# 02 WBS 文档

> 要求：WBS 必须围绕交付物（成果）分解，至少分解到三级。

## 1. 一级交付物
本组 WBS 以最终需要提交和检查的成果为主线，而不是以“讨论、开会、写代码”等过程动作作为主线。一级交付物划分如下：

1. 项目管理成果
2. 范围计划成果
3. 进度计划成果
4. 配置管理成果
5. 示例代码成果
6. 总结报告成果

其中，项目管理成果用于说明项目背景、目标、角色和边界；范围计划成果用于明确三级 WBS；进度计划成果用于安排关键活动；配置管理成果用于保证 GitHub 协作过程可追溯；示例代码成果只作为项目结构支撑，不作为本实验重点。

## 3. 分解说明
WBS 分解遵循交付物导向原则，优先保证文档成果完整、版本记录清楚、责任分工明确。
