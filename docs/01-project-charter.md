# 01 项目启动说明

## 1. 项目名称
校园活动报名与签到管理系统

## 2. 项目目标
- 支持活动发布
- 支持学生报名
- 支持签到与统计

本项目面向校内活动组织与参与场景，目标是在不开发完整业务系统的前提下，完成项目管理过程中的关键成果，包括项目启动说明、三级 WBS、简化进度计划、配置管理方案、冲突解决记录和总结报告。
从项目管理角度看，本组重点关注交付物是否清晰、责任人是否明确、版本记录是否可追溯。系统功能只进行象征性 Java 示例实现，主要实验成果集中保存在 `docs` 文件夹中。

## 3. 团队分工
吴宇轩担任项目经理和仓库负责人，负责创建 GitHub 仓库、建立项目目录、维护 `main` 与 `dev` 分支，并在最终阶段将全组成果合并到 `main` 分支。

## 4. 项目边界
本实验不要求完成完整系统开发，不进行真实部署，不接入学校统一身份认证系统，不实现高并发优化和移动端 App。

## 5. 主要交付物
| 交付物 | 验收标准 |
|---|---|
| 项目仓库 | 目录结构完整，包含 `README.md`、`docs`、`src`、`test`、`assets` |
| WBS 文档 | 围绕交付物分解，至少达到三级，不以过程动作作为主线 |
| 进度计划文档 | 包含活动名称、工期、前置关系、责任人和对应交付物 |
| 配置管理文档 | 明确配置项、分支策略、提交规范、合并规则和冲突处理方式 |
| 总结报告 | 由全体成员共同编辑，说明组织过程、WBS、进度、配置和冲突解决 |
