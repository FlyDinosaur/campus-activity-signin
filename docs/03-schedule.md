# 03 简化进度计划

## 1. 总体里程碑

| 里程碑编号 | 里程碑名称 | 预计完成时间 | 完成标准 |
|---|---|---|---|
| M1 | 仓库初始化完成 | 第 1 天上午 | `main` 与 `dev` 分支创建完成，目录结构完整 |
| M2 | 基础文档框架完成 | 第 1 天上午 | `docs` 文件夹下 6 个 Markdown 文档创建完成 |
| M3 | 成员分支提交完成 | 第 1 天下午 | 四名成员均完成个人分支提交 |
| M4 | 冲突解决完成 | 第 1 天下午 | `docs/conflict.md` 冲突被识别并手工解决 |
| M5 | 最终成果合并完成 | 第 1 天下午 | `dev` 合并至 `main`，最终报告位于主分支 |

总体进度采用“先结构、后内容、再集成”的安排。吴宇轩先完成仓库与文档框架，其他成员依次补充 WBS、进度和配置内容，最后统一合并。
